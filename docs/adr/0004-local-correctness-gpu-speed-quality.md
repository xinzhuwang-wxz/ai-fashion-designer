# 两阶段验收边界：本地验正确性，租卡后交付速度与画质

## Status

accepted

## Context

M4 MPS 一次扩散重绘需几十秒，物理上做不到截图里 0.5–1s/笔、30–60fps 的实时体验。用户的最终目标是达到截图效果。若在本地硬追实时与最高画质，会卡死在调参与发热里、进度停滞，且容易让期望落空。

## Decision

把交付明确分成两期：

- **本地（M4）阶段 = 验"对不对"（正确性与完整链路）**：六场景串通、选变体真正生效、局部编辑只改 mask、双画布、草图优先入口、下载 / 参数导出 / 项目持久化都可用。用合成图与少量真实服装图验链路，不追帧率。
- **租卡（GPU）阶段 = 交付"快不快 / 美不美"**：StreamDiffusion 实时预览层、IP-Adapter 保原款风格、BrushNet/PowerPaint 局部重绘、1024×1536+ 基础生成与高清放大。因接口与工作流解耦，租卡后基本只需"换工作流 + 提分辨率"即可逼近截图。

## Consequences — 本地阶段明确不承诺

- 30–60fps AI 生成；每笔 0.5–1s 语义正确结果；
- 复杂蕾丝/褶皱精准保留；mask 外绝对零变化（本地 inpaint 边界可能有轻微过渡）；
- 最终图直接用于生产；上传到成品全程 1–3 分钟。

这些指标归入租卡后阶段，且需经真实服装数据集测试后才对外承诺。
