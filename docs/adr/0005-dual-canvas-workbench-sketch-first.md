# 双画布工作台 + 草图优先入口 作为核心交互（前端重写）

## Status

accepted

## Context

旧前端是"单 Tldraw 画布 + 右侧被动预览面板"，与目标产品（抖音截图所示）的"左线稿可编辑 / 右成衣渲染"并排工作台差距最大，体验上不成立；且只有"上传照片"一个起点，缺少截图与场景里同等重要的"直接画/传线稿出成衣"入口。

## Decision

重写前端为真·双画布工作台：

- **左画布**：可编辑的线稿/草图画布，承载画笔、上传、手绘；复用 Tldraw 作矢量编辑轮子；负责笔触与"图像归一化坐标"输出（喂 ADR-0003 的局部编辑）。
- **右画布**：只读的成衣渲染画布，显示 `Material` / `Edit Version` / `Final`；配 seed、示例、材质文本输入栏（对应截图底栏）。
- **草图优先入口**：左画布空手起笔直接画线稿、或上传线稿，即可出成衣，不强制先上传照片；与"上传照片"链路在 `Lineart` 节点汇合（见 ADR-0002）。

## Consequences

- 前端从单画布形态整体重写；左右职责分明（左编辑、右只读渲染）。
- 两个起点在 UI 上并列，草图优先成为一等入口而非特例。
